정규식(Regular Expression) : 문자열에서 특정 내용을 찾거나 대체, 발췌하는데 사용

  • regex 단어로 많이 표현됨.
  • 특정 데이터를 탐색, 확인, 존재여부 등을 쉽게 확인하기 위한 규칙
  • 웹 분야에서는 password 작성규칙 , 첨부파일... 자주쓰는 사용X
  • 데이터 분석과 관련된 분야해서는 반드시 꼭 알고 있어야 함.
  • 알고있으면 편리한 기능. 다른 언어에서도 거의 비슷한 형식이므로 한번 학습해 놓으면 유용하게 사용
  • /regexr/i : (/ /) : 시작, 종료기호, (regexr) : pattern, (i) : flag
  • ("str").match(/regexr/flag) : str에서 매칭되는 항목들을 배열로 반환
  • ("str").replace(/regexr/, "바꿀문자열") : 정규표현식에 매칭되는 항목을 바꿀문자열로 변환
  • ("str").split(/regexr/) : 문자열을 정규표현식에 매칭되는 항목으로 쪼개어 배열로 반환
  • (정규표현식).test("str") : 문자열이 정규표현식 규칙에 맞으면 true, 아니면 false
  • 정규표현식 플래그
  • i : 대/소문자 구분X
  • g : 전체 검색
  • m : 행바꿈도 포함
  • a-zA-Z : 영어 알파벳 / ㄱ-ㅎ가-힣 : 한글 / 0-9 : 숫자
  • . : 모든문자열(숫자, 한글, 영어, 특수기호, 공백 모두) 줄바꿈X
  • \d :숫자 \D : 숫자 아닌것 \s :공백(space) \S : 공백 아닌것 \w : 영숫자 _ \W : \w 아닌것 \특수기호 : \!\@ ...
  • 검색 패턴 : | OR a|b
  • [] : 괄호안의 문자들 중 하나 or처리의 묶음으로 보면 됨 /abc/ => "abc"를 포함하는 /[abc]/ => "a" 또는 "b" 또는 "c" 를 포함하는 [다-바] => 다, 라, 마, 바
  • [^문자] : 괄호안의 문제를 제외한 것 [^abcd] : "a","b","c","d" 4개 문자를 제외
  • ^문자열$ : ^(시작) $(끝)
